# .team/charter.md — 상담 결론의 정본 (뼈대)

> 언어·항목은 대상 서비스에 맞게 조정. 결론만이 아니라 **근거**(기각된 대안, 원칙 이탈과 그 사유)를 남긴다 — 재상담은 이 문서와의 diff로 이루어진다.

## 서비스

- 무엇을 하는 서비스인가 / 누가 쓰나 / 수익 모델 (참고용 — goal에는 안 들어감)
- 스택, 실행/테스트/배포 방법

## 팀 구성

역할별로: 이름, 하는 일, **하지 않는 일**, agent 파일 경로. 이 구성을 택한 근거와 기각한 대안.

## 일하는 방식

- **실행 모델**: 어떤 역할이 어떤 주기의 루틴으로 도는지 / 무엇은 사람이 트리거하는지 / 존재하는 커맨드와 그 이름 (등록된 루틴의 스케줄 명세 포함)
- **goal 규약**: 조건은 산출물 기준 (테스트 통과, 승인 파일 존재 — "개선됨" 금지)
- **백로그**: `.team/backlog.md`, 항목마다 goal 조건 필수 — 없으면 발행 불가
- **게이트**: 사람 승인 필수 목록 (외부 발행 / 프로덕션 배포 / 과금 변경 / …), 대기물은 `.team/gates/`에 축적, 사람이 확인하는 리듬

## 지표와 예산

- **선행 지표** (주기 평가용): 무엇을, 어디서 측정
- **후행 결과** (매출 등): 사람 확인, 느린 주기 — goal·주기 평가와 분리 (3층 구조)
- **토큰/API 예산**: 월 상한, 확인 데이터 소스

## 센서 현황과 첫 백로그

지표 수집 / 피드백 인입 / 배포 게이트 — 있는 것과 없는 것. 없는 것이 첫 백로그.

## 원칙 이탈 기록

사용자가 명시적으로 원칙을 넘어서기로 한 결정과 그 사유. (없으면 "없음")
