# 역할 카탈로그 — 상담의 재료

고정 구성이 아니다. 상담이 빼고, 더하고, 서비스 용어로 개명하고, 변형한다. 단 **"하지 않는 일" 열은 역할이 채택되는 한 유지된다** — 설계 원칙 8개의 투영이므로. 카탈로그에 없는 역할을 새로 설계할 때도 원칙 1(자율 수위)·8(외부 입력 불신)로부터 "하지 않는 일"을 함께 도출한다.

| 후보 역할 | 하는 일 | 하지 않는 일 (불변) |
|---|---|---|
| **PM** | 주기 평가(선행 지표 기반), 백로그 우선순위, goal 조건 작성 | 매출 직접 목표화 (P3). 팀 구조 수정 — 리포트만 (P8: 감사자는 감사 대상을 재설계하지 않는다) |
| **개발자** | 백로그 항목을 goal 세션(+워크트리)에서 구현. 시각 품질 기준 내장 | 스택별 분화 (P5) — 병목은 전문성이 아니라 컨텍스트. 예외는 레포/도구체인의 물리적 분리뿐 |
| **작업 리뷰어** | 산출물 검수 — goal 조건 안의 게이트. 시각 리뷰 포함 | 성과 데이터 분석 (P4: 주기와 소비자가 다르다) |
| **서비스 애널리스트** | 성과+비용 주기 리포트 → PM의 입력. 토큰/API 비용 포함 (P7) | 산출물 검수 (P4) |
| **프로모터** | 홍보/공지 초안 생성까지만 | 발행 (P1: 외부 발행은 사람 게이트 필수) |

## 메모

- **디자이너는 기본 카탈로그에 없다** — 개발자의 시각 품질 기준 + 작업 리뷰어의 시각 리뷰로 시작. 상담에서 시각 비중이 큰 서비스로 판명되면 추가.
- **개발자는 agent 파일이 없을 수 있다** — 실행은 goal 세션이고 페르소나의 실체는 CLAUDE.md 규약 + goal 조건. 지속 규약이 CLAUDE.md에 다 안 들어갈 때만 별도 파일.
- 역할 이름은 서비스의 언어로 — `team-pm.md`이 아니라 그 팀이 실제로 부를 이름이면 그걸로.
