# Project: claude-sync

> **Frozen Core.** 변경 시 Heavy 모드 — 팀 전원 합의 필요.

## 한 줄 정의

여러 개발자가 각자의 LLM 에이전트로 동시에 개발할 때 발생하는 협업 문제를 해결하는 MCP 기반 동기화 레이어.

## 유저 시나리오 (핵심)

1. **개발자가 새 작업을 시작할 때** — 자신의 에이전트가 "다른 팀원의 진행 중 작업, 관련 ADR, 영향받을 영역"을 자동으로 알려준다.

2. **두 개발자가 같은 영역/함수에 손을 댈 때** — 코드 한 줄 짜기 전에 에이전트가 충돌을 감지하고 슬랙으로 조율을 시작한다.

3. **개발 세션 중 결정이 발생할 때** — 에이전트가 결정 후보를 자동 감지하고, 세션 종료 시 사람이 1초 안에 확정/거부한다.

4. **누군가 과거 결정을 뒤집을 때** — 에이전트가 영향받는 다른 ADR/코드/사람을 자동 식별하고 reversal 워크플로우를 안내한다.

5. **신규 멤버가 합류해 위험한 영역에 들어갈 때** — 에이전트가 관련 정책 ADR을 자동 제시하고, 필요 시 보안/PM 조율을 트리거한다.

## 시스템 경계

**우리가 만드는 것:**
- MCP 서버 (claude-sync 코어)
- `.sync/` 디렉토리 데이터 모델 (Intent / Decision / Coordination / Migration / Retrospective / Draft)
- 결정 감지 / Reversal 감지 / 영향 분석 / 모드 판단 로직
- Slack 봇 (Active Coordination 입출구)
- IDE/에이전트 통합 (Claude Code hooks 우선, 이후 다른 MCP 호환 에이전트)

**외부 의존:**
- MCP 프로토콜
- Git (브랜치 prefix로 모드 판단, .sync/ 저장)
- Slack API (Coordination)
- (선택) Calendar API, PR/Issue 트래커 API

**우리가 만들지 않는 것:**
- 새로운 IDE / 에디터
- 새로운 메신저 / 채팅
- 일반 PM 도구 (Linear/Jira 대체 X)
- 에이전트 ↔ 에이전트 자동 통신

## 비기능 요구사항

- **로컬 우선** — 핵심 동작은 로컬 MCP 서버. 외부 서비스 의존 최소화 (프라이버시).
- **모든 에이전트 도구 호환** — MCP 표준 준수. 특정 IDE 락인 X.
- **점진적 도입** — 모든 기능 활성화 강요 X. 팀이 필요한 부분부터 켤 수 있어야.
- **빠른 응답** — 에이전트 hook이 사용자 경험을 느리게 만들지 않아야 함. 충돌 검사 등은 백그라운드.
