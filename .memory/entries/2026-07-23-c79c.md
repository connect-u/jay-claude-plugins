2026-07-23 Jay 결정 (v0.2.1). 에이전트 대면 텍스트(manifest 기록 규약, 훅 블록 메시지, 서브에이전트 주입문, MCP 툴 설명·결과)는 영문 고정. 저장 엔트리(title·body)와 사용자 영수증의 언어는 .memory/config.json의 language 키(자유 문자열, 기본 "English")로 프로젝트별 설정. 이 repo는 "Korean" — 기존 한국어 엔트리와의 일관성.

근거: 지시문 영문은 LLM 준수율·이식성 (커맨드명·지시문 영문 / 산출물 로컬 언어 이원화 원칙과 같은 계보). 저장 언어를 프로젝트별 고정값으로 두는 이유: memorj 검색이 grep 기반이라 저장소에 언어가 섞이면 질의 언어와 다른 엔트리를 영원히 못 찾는다 — 언어 일관성이 곧 검색 recall. "대화 언어 따라가기(auto)"는 이 이유로 기각.

같이 도입된 기록 영수증: memory_write 직후 📝 블록([type] title (id) + 한 줄 요지)을 설정 언어로 표시 — recall 지향 포착의 사람 쪽 감시 루프 (이상 기록을 입구에서 교정, 불변식 3). 트리거가 "기록 발생" 사건이라 insight류 큐레이션과 달리 재량이 없다.
