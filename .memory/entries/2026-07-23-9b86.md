jay-claude-plugins 마켓플레이스는 로컬 작업 사본이 아니라 git 원격(github connect-u/jay-claude-plugins)을 소스로 클론된 사본이다 (~/.claude/plugins/marketplaces/, known_marketplaces.json의 source가 git URL). 따라서 plugin.json 버전을 올리고 로컬에 커밋해도 push 전에는 "already at the latest version"이 나온다 — memorj 0.2.1 배포에서 실증 (마켓플레이스 업데이트·플러그인 업데이트를 눌러도 0.1.0 고정).

반영 전체 경로: (1) 로컬 커밋 + plugin.json 버전 범프 → (2) git push origin main → (3) claude plugin marketplace update <이름> (또는 /plugin의 마켓플레이스 업데이트) → (4) /plugin에서 플러그인 update → (5) /reload-plugins. 캐시 사본 실행 원리(ef73)의 상류 단계. 참고: 카탈로그(.claude-plugin/marketplace.json)에는 버전 필드가 없고 각 플러그인의 plugin.json이 버전 정본 — 카탈로그 설명에 버전 문자열을 하드코딩하지 말 것.

부수 발견: 원격에 다른 작업(PR 머지)이 먼저 들어가 있으면 .memory/epoch이 rebase 충돌 지점이 된다 — 카운터 파일이라 해결은 큰 값 채택.
