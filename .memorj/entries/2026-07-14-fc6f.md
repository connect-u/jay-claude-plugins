에이전트 팀(PM·리뷰어·작업자로 부업 서비스를 자율 개선)을 플러그인화할 때, 팀 런타임 프레임워크(가칭 jrew)가 아니라 **팀 생성기(build-agent-team)**로 만들기로 결정.

근거: 생성기의 산출물은 대상 레포의 평문 파일(.claude/agents/*.md, commands, CLAUDE.md 규약 섹션)이라 추상화를 틀려도 수정 비용이 낮다. 런타임은 Claude Code 순정 기능(/goal, agents, routines)과 memorj가 담당하므로 프레임워크를 새로 만들 자리가 없다. 서비스마다 다른 부분은 생성 시점의 인터뷰(상담)가 수집하고, 공통 설계 원칙은 생성기가 내장한다.

기각된 대안: (a) 팀 런타임 플러그인 — 한 번도 안 굴린 팀의 일반화로 추상화 오류 위험, (b) 부업 레포에 직접 세팅(플러그인 없이) — 생성기 방식이 이를 포괄하며 지식을 재사용 가능하게 함.

이름 확정: 메모리 플러그인 engram → **memorj**로 개명(2026-07-14, 커밋 d0709a7), 팀 생성기는 **build-agent-team**. Jay의 작명 취향은 말장난 계열(J-라인: jarness, memorj — 한 글자를 j로 치환).

진행 순서 합의: 설계 문서 초안(인터뷰 항목 + 생성물 템플릿 뼈대) → 검토 → 구현 → Jay의 부업 서비스에서 첫 실행 (서비스 정보는 인터뷰가 그때 수집).
