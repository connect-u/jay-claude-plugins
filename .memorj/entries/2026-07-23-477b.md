세 프로젝트(jarness, cc-git, claude-sync) 별도 개발 중단 확정. 처리 방식 두 가지 대안 중 선택:

- 기각: 폴더는 제자리에 두고 marketplace.json에서만 제거 — 폴더가 죽었다는 사실을 알려면 marketplace.json 교차 확인이 필요해서. 상태는 대상이 사는 자리에서 선언돼야 한다(deprecated는 정지 표지판 원칙과 동형).
- 채택: deprecated/ 폴더로 git mv + 마켓플레이스에서 cc-git·jarness 제거. 루트는 살아있는 것의 목록(memorj, build-agent-team)으로 유지. git mv라 히스토리 보존.

삭제하지 않는 이유: 아이디어 계보가 살아있는 프로젝트의 왜를 설명한다 — claude-sync의 "결정은 발견되는 것" 철학 → memorj 포착 규칙, jarness의 상담 방식 → build-agent-team. deprecated/README.md에 계보 표 기록. claude-sync-design.md(v1)는 deprecated/claude-sync/docs/design-v1.md로 이동.
