~/.memory/.state/projects/&lt;slug&gt;.json은 세션이 아니라 프로젝트 slug 키다. 같은 프로젝트 디렉토리에서 세션 두 개를 동시에 돌리면 서로의 stop_blocks 카운터와 settled 플래그를 덮어쓴다 — 한 세션의 settle/리셋이 다른 세션의 fail-open 상한을 무력화하거나, SessionStart의 state 초기화가 진행 중인 다른 세션의 카운터를 지운다. 미정산 경고(직전 세션 미정산 감지)도 동시 세션에서는 오판 가능.

Jay의 워크플로(worktree + 메인 세션 병행)에서 실제로 발생 가능한 조건. worktree는 root 경로가 달라 slug가 갈리지만, 같은 디렉토리의 두 탭이면 충돌한다. 수정 시 state 키를 session_id 단위로 바꾸거나 세션별 필드로 분리하는 것을 검토할 것.
