2026-07-23 결정 (v0.3.0). 설정 읽기는 프로젝트 .memorj/config.json → 사용자 ~/.memorj/config.json → 내장 기본값의 폴백 체인. 언어 같은 사용자 성향을 프로젝트마다 수동 생성 요구하는 것은 보장 1(캡처에 사람의 규율 0)과 충돌 — 사용자 기본값에 한 번 세팅하면 전 프로젝트 적용, 프로젝트 파일은 "git 공유 저장소의 언어 고정" 오버라이드 용도만.

UX는 /memorj:setting 슬래시 커맨드: 인자 없이 = 현황 표시(값+출처) + 초기 세팅 유도, "key value [--project]" = 변경. 키는 확장 가능(현재 language뿐) — language 단일 커맨드가 아니라 setting으로 이름 지은 이유. 커맨드 지시문은 영문 (v0.2.1 언어 이원화 원칙), argument-hint + $ARGUMENTS 방식.

맥락: 처음엔 프로젝트 파일만 있어서 "새 프로젝트마다 직접 만들어야 하나?"라는 Jay 질문이 결함을 드러냈다. Jay가 커맨드 UX를 제안했고, 폴백 체인(읽기 규칙)과 커맨드(쓰기 UX)는 상보적이라 결합.
